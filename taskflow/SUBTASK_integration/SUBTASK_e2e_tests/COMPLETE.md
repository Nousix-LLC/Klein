## How the suites map to `examples/index.json`

Everything is **data-driven** — the suites iterate the manifest and hard-code no
per-example expectation, so adding an example (with its generated goldens) extends
coverage automatically. The manifest currently holds **10 entries** (6 `ok`, 4 `error`):

- **`ok` (6)** — `fibonacci`, `fizzbuzz`, `closures`, `higher_order`, `data_structures`,
  `strings`. Each is run through `interpret(source, { sourceName, write: capture })`;
  the suite asserts `outcome.ok === true`, `outcome.diagnostics === []`, and captured
  stdout **byte-exact equals** `examples/<name>.out`.
- **`error` (4)** — `errors_undefined` (`[E3001]`), `errors_type` (`[E3002]`),
  `errors_divzero` (`[E3009]`), `errors_syntax` (`[E2003, E2003]`). Each is run through
  `interpret()`; the suite asserts `outcome.ok === false`, then
  `diagnostics.map(d => d.code)` **deep-equals** the manifest's ordered `expectedCodes`
  (structured — never message text). It further asserts every diagnostic carries a
  `span` (real numeric offsets), and that rendering via the shared `@core`
  `formatDiagnostic(d, source, { color:false })` yields a source snippet, a `^` caret,
  and a `--> name:line:col` location — and that the full rendered block is **byte-exact
  equal** to the committed `examples/<name>.diag` golden.

## True-process end-to-end (the `bin` path, not just the facade)

`process.test.ts` spawns the shipped executable with `spawnSync(node, ["bin/klein.mjs",
"examples/<name>.kl"])` from the project root, `NO_COLOR` set and `FORCE_COLOR` stripped
(so the child's stderr is clean and color-free):
- `fibonacci` — asserts exit `0`, stdout equals `.out`, empty stderr.
- `errors_undefined` — asserts exit `1` (diagnostics), stdout equals `.out`, and stderr
  equals the `.diag` golden (the CLI's stderr is exactly how the golden was generated),
  plus the structured `error[E####]` + `-->` markers.

Kept to two spawns deliberately (tsx cold-start is not free); the in-process suites
cover every manifest entry exhaustively. Spawns carry a 30s timeout for tsx startup.

## Design notes / fidelity decisions

- **Golden source name.** In-process runs pass `sourceName = "examples/<name>.kl"` so the
  rendered `--> …:line:col` locations and runtime stack frames match the goldens, which
  the `examples` task generated by invoking the CLI with that exact path argument.
- **CLI-faithful diagnostic reconstruction.** `renderDiagnostics` mirrors `runSource` in
  `src/cli/cli.ts` byte-for-byte (`formatDiagnostic(...) + "\n"` per diagnostic, in
  source order), so the in-process `.diag` comparison equals the real CLI stderr without
  a second child-process spawn per error example.
- **Style/quality.** Suites import the public surface from `../../src/index` and the
  renderer from `@core`; key off `ErrorCode`/structure, never message text; match the
  existing `tests/**` Vitest + injected-I/O conventions. `harness.ts` has no `*.test.ts`
  suffix so Vitest never collects it as a suite (mirroring `tests/stdlib/harness.ts`).
